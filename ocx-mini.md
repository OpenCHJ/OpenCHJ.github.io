# OCX mini 文書定義

（人文系向け説明版・Draft）

## 1. OCX mini とは何か

OCX mini は、文学作品・史料・資料テキストを\
**形態素解析・検索・注釈付けに使いやすくするための、最小限のマークアップ方式**です。

最大の特徴は次の2点です。

1.  **本文に現れる文字はすべて解析対象になる**
2.  **解析に不要な情報（読み・注記・書誌情報）は本文から分離する**

これにより、

-   解析結果を **文字オフセット**で安定して管理できる
-   元の本文文字列を壊さずに、注釈や読みを追加できる
-   XHTML 文書にも、プレーンテキストにも同じ考え方で使える

という利点があります。

## 2. 文書の書き方（2つの使い方）

### (A) プレーンテキスト最小マークアップ（推奨）

``` xml
<ocx:doc textID="江戸川乱歩　二銭銅貨"
         corpusName="青空文庫"
         xmlns:ocx="https://openchj.github.io/ns/ocx">
二銭銅貨<ocx:eos/>
江戸川乱歩<ocx:eos/>
「あの泥坊が羨しい」二人の間にこんな言葉が交される程、其頃は窮迫していた。<ocx:eos/>
</ocx:doc>
```

### (B) XHTML 文書への埋め込み

``` xml
<body>
  <ocx:doc textID="CHJ0001" corpusName="青空文庫">
    あの泥坊が羨しい<ocx:eos/><br/>
  </ocx:doc>
</body>
```

## 3. OCX mini の基本原則

### 本文原則

`<ocx:doc>` 内の文字はすべて形態素解析対象。

### 文境界

文末には必ず `<ocx:eos/>` を置く。

## 4. 使用可能な要素

-   `<ocx:doc>`
-   `<ocx:eos/>`
-   `<ocx:r>`
-   `<ocx:note/>`
